# Bucky Agent Startup Order

버키에이전트는 AEONID Agent Brain OS의 작업 착수 전 지휘자 역할을 한다.

Codex와 Claude Code는 작업 시작 전에 이 문서를 읽고, 아래 순서를 따른다.

## Mission

프로젝트를 실제 실행 가능한 상태로 유지하면서, 각 PC가 정해진 브랜치 역할 안에서 개발을 이어가게 한다.

## Operating Principles

1. GitHub `origin`을 중앙 기준으로 사용한다.
2. `main`은 안정 버전과 병합 기준으로만 사용한다.
3. 노트북 PC는 `laptop/main-dev`에서만 개발한다.
4. 사무실 PC는 별도 `office/...` 브랜치를 사용한다.
5. 집PC는 검토, 병합, 안정 버전 확인을 우선한다.
6. 작업 시작 전 `git status`와 현재 브랜치를 확인한다.
7. 작업 종료 전 실행 가능성 또는 빌드 가능성을 검증한다.
8. API key, token, password, 개인정보는 Git에 올리지 않는다.
9. 불필요한 라이브러리와 과한 구조 변경은 피한다.
10. 실제 작동 여부를 설계보다 우선한다.

## Required Startup Checklist

작업자는 아래 항목을 확인하고 보고한 뒤 수정에 들어간다.

- 현재 프로젝트 목적
- 현재 브랜치
- 변경할 파일
- 실행 명령
- 빌드 명령
- 민감정보 또는 배포 위험 여부

## Required Finish Checklist

작업 완료 후 아래 항목을 보고한다.

- 수정된 파일
- 변경 요약
- 실행/빌드 검증 결과
- 남은 위험
- 커밋/푸쉬 상태
- 집PC에서 이어받는 명령

## Decision Rule

작업자가 판단하기 어렵거나 위험한 변경이면 바로 구현하지 말고 먼저 보고한다.

위험한 변경 예시:

- 인증 구조 변경
- 데이터베이스 스키마 변경
- 배포 설정 변경
- API key 처리 변경
- 파일 업로드 보안 변경
- `main` 브랜치 직접 수정
